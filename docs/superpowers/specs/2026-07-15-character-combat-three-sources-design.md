# 阶段 3 角色战斗与三源设计

**日期：** 2026-07-15  
**状态：** 已确认  
**上游依赖：** 阶段 2 输入与自由相机已通过自动化、构建和真机验收  
**范围：** 四段普攻、闪避与体力、三源技能与反应、共鸣终结技、训练假人与训练脉冲

## 1. 目标与边界

本阶段在固定步循环、类型化事件、双区触控、自由相机和软锁定基础上，交付一套可确定性
测试、可在真机操作、可被后续敌人和首领复用的角色战斗域。

阶段 3 完整交付以下能力：

- 四段普攻及移动、受击、超时重置。
- 消耗体力的闪避、无敌帧、精准闪避和“源流洞察”。
- 辉印、脉流、蚀质三个源技能及其冷却。
- 折光、凝滞、崩解三种双源反应，以及限时三源连携。
- 共鸣能量和共鸣终结技。
- 具备生命、韧性、附着和状态效果的静态训练假人。
- 只含预警和命中窗口的固定节奏训练脉冲。
- 六个临时可见战斗按钮和完整诊断快照。

以下内容明确不属于阶段 3：

- 敌人感知、决策、追踪、技能选择和移动，统一留到阶段 4。
- 首领阶段、遭遇流程和关卡门，统一留到阶段 5。
- 正式 HUD 布局、动画、VFX、音频、震动和性能降级，统一留到阶段 6。
- 脉流的真实牵引、折光的多目标传导。当前单假人环境只保留确定性数值和状态语义。

## 2. 架构选择

采用“独立战斗域为核心，事件用于跨系统通知”的混合结构。

战斗状态和数值按固定顺序由小型 Native C++ 模块直接处理，避免完全事件驱动产生递归事件、
隐含优先级和难以复现的同 tick 行为。战斗域仍输出 `GameplayEvent` 和
`PresentationEvent`，供 HUD、渲染和后续音效消费；消费者不得反向修改战斗状态。

单个固定 tick 的处理顺序为：

1. 恢复体力并推进冷却、附着和限时窗口。
2. 从有序输入队列规范化一次性战斗动作。
3. 推进角色动作状态并生成攻击或技能命中请求。
4. 推进训练脉冲，判定普通命中、无敌或精准闪避。
5. 按稳定顺序结算生命伤害和韧性伤害。
6. 更新源附着，解析至多一次双源反应和三源连携。
7. 更新共鸣、破韧、死亡和训练重置状态。
8. 发布游戏事件、表现事件和线程安全快照。

事件按 `(tick, attacker, target, sequence)` 稳定排序。当前 tick 产生的派生事件进入下一处理
批次，不在同一遍遍历中递归消费。

## 3. 组件设计

### 3.1 `CombatConfig`

集中保存连击窗口、各段伤害、体力、冷却、无敌帧、附着时长、反应、共鸣、训练假人和
训练脉冲参数。配置使用强类型结构，不以字符串键驱动战斗逻辑。

配置校验按相关字段分组。任一分组出现非有限值、负时长、倒置窗口、非正上限或不合法倍率
时，该分组整体回退到安全默认值，避免部分回退形成互相矛盾的规则。

### 3.2 `ActionStateMachine`

管理待机、普攻一至四段、闪避、三个源技能和终结技。一次性输入只在允许的开始、取消或
衔接窗口消费。移动、受击或连击窗口超时会重置普攻段数。

同一 tick 收到多个互斥动作时按输入 `sequence` 处理，第一个成功进入状态的动作获胜，其余
动作得到明确拒绝原因，不覆盖当前动作。

### 3.3 `CombatResources`

管理体力、体力恢复延迟、技能冷却、源流洞察、共鸣能量和终结窗口。所有时间使用整数
tick 或整数毫秒，所有数值沿用 `FixedPoint`。

### 3.4 `DamageResolver`

接收规范化命中请求，计算生命伤害、韧性伤害、无敌、易伤、破韧和死亡，并返回确定性的
结算结果及事件。它不读取 ArkTS 状态，不播放表现，也不选择 AI 动作。

### 3.5 `SourceReactionSystem`

管理训练目标的源附着、刷新、过期、双源反应、三源连携和重复触发保护。同源命中刷新该源
的持续时间与强度，不把同源视为反应。一次命中最多触发一种双源反应；组合结果与输入顺序
无关。

### 3.6 `TrainingTarget`

训练假人占用当前可软锁定目标，具有位置、生命、韧性、源附着、弱点、凝滞、侵蚀、破韧
和死亡重置状态。它不会移动、攻击或做决策，因此不构成敌人 AI。

### 3.7 `TrainingPulse`

按固定 tick 周期经历空闲、预警、命中和间隔状态。命中只用于验证玩家受击、无敌帧、精准
闪避和连击重置；它不追踪角色、不选择技能，也不改变训练假人的行为。

### 3.8 `CombatController`

只负责按固定顺序编排以上模块、收集拒绝原因和发布结果。具体伤害、窗口、资源和反应规则
不得堆积在控制器中。

### 3.9 输入、桥接与 HUD

ArkTS 在右下区域提供普攻、闪避、辉印、脉流、蚀质和终结技六个临时可见按钮。按钮只调用
`pushAction(action)`，由 N-API 校验枚举后写入与触控共享的有序输入队列；按钮不得直接修改
Native 战斗状态。

临时按钮优先保证可操作、可区分和不遮挡既有移动/相机区域。正式布局、图标、冷却遮罩和
动效留到阶段 6。

### 3.10 快照与事件

`GameSnapshot` 增加：

- 玩家生命、韧性、体力和当前动作。
- 当前连击段、连击剩余窗口和无敌状态。
- 三个技能冷却、源流洞察剩余时间。
- 共鸣能量、终结窗口和最近动作拒绝原因。
- 假人生命、韧性、破韧状态和三源附着。
- 当前反应、训练脉冲阶段及预警剩余时间。

游戏事件覆盖命中、伤害、受击、闪避、精准闪避、破韧、附着、反应、共鸣充满、终结技、
死亡和训练重置。表现事件只描述闪光、镜头、破韧和共鸣等表现意图，不持有游戏规则。

## 4. 初始规则与数值

这些数值用于测试和真机验收，不代表正式平衡。

### 4.1 普攻与闪避

| 项目 | 初始值 |
|---|---:|
| 四段普攻生命伤害 | `8 / 10 / 12 / 18` |
| 四段普攻韧性伤害 | `4 / 5 / 6 / 10` |
| 连击衔接窗口 | 命中后 `480ms` |
| 体力上限 | `100` |
| 闪避消耗 | `30` |
| 体力恢复延迟 | `500ms` |
| 体力恢复速度 | `20/秒` |
| 闪避总时长 | `300ms` |
| 无敌时长 | 闪避开始后的 `200ms` |
| 精准闪避窗口 | 训练脉冲命中前剩余 `100–500ms`（含边界） |
| 源流洞察持续时间 | `15 秒` |
| 洞察增益 | 下一源技能伤害和附着量 `+50%` |

移动输入非零、玩家实际受击或 `480ms` 衔接窗口过期时重置连击。只有成功命中的源技能才会
消费源流洞察；因无目标、冷却或动作冲突被拒绝时不消费。

精准闪避只在当前训练脉冲尚未命中且剩余时间位于闭区间 `[100ms, 500ms]` 时成立；
`501ms`、`99ms` 以及命中点之后均按普通闪避处理。精准闪避成功后直接规避其对应的本次
训练脉冲伤害并获得源流洞察，不依赖普通闪避开始后 `200ms` 的无敌区间。该规则避免玩家
在窗口前段成功触发洞察，却因无敌时间先结束而仍被同一次脉冲命中的矛盾结果。

### 4.2 三源技能与反应

| 技能 | 冷却 | 主要语义 |
|---|---:|---|
| 辉印 | `3 秒` | 生命伤害 `20`、韧性伤害 `6`，施加辉印并暴露弱点 |
| 脉流 | `4 秒` | 生命伤害 `16`、韧性伤害 `8`，施加脉流并准备凝滞 |
| 蚀质 | `5 秒` | 生命伤害 `12`、韧性伤害 `18`，施加蚀质和侵蚀 |

源附着默认持续 `6 秒`。反应规则为：

- 辉印 + 脉流产生折光：追加 `12` 生命伤害，并施加 `3 秒`、承伤 `+20%` 的弱点。
- 脉流 + 蚀质产生凝滞：施加 `4 秒` 凝滞，使后续韧性伤害 `+50%`。
- 辉印 + 蚀质产生崩解：立即追加 `30` 韧性伤害；若触发破韧，再追加 `20` 生命伤害。

反应消费参与组合的两种旧附着，再应用当前命中的新附着结果，确保同一对附着不会被普通
连点重复触发。每次源技能命中增加 `10` 共鸣，每次双源反应额外增加 `20` 共鸣。

### 4.3 三源连携与终结技

任意 `8 秒` 窗口内成功命中三种不同源即完成三源连携，立即把共鸣充至 `100` 并开启
`5 秒` 终结窗口。普通源技能和双源反应也按配置逐步增加共鸣，但不得超过 `100`。

终结技只在共鸣为 `100` 时可释放。成功释放后消耗全部共鸣，关闭终结窗口，并对当前有效
目标造成 `60` 生命伤害和 `40` 韧性伤害。窗口过期不会清空已经充满的共鸣，但会移除本次三源连携提供
的窗口增益；这样普通积累仍可最终释放终结技。

### 4.4 训练环境

- 训练假人初始为 `HP 300 / 韧性 100`。
- 破韧后进入 `3 秒` 易伤并承受 `+25%` 生命伤害，结束后韧性恢复至 `100`；死亡 `2 秒`
  后自动重置全部临时状态。
- 玩家训练状态初始为 `HP 100 / 韧性 100`。训练脉冲每 `3 秒` 发生一次，其中预警
  `800ms`、命中伤害 `10`。玩家处于无敌帧时不受伤；满足精准窗口时额外获得源流
  洞察；普通受击会扣除生命并重置连击。
- 玩家生命降至零时进行训练复位，不进入正式失败或重试流程。

## 5. 拒绝、生命周期与安全

以下情况拒绝动作且不部分修改状态：

- 当前没有有效目标，或目标已经死亡。
- 技能仍在冷却。
- 闪避体力不足。
- 终结技能量不足。
- 当前动作不允许被新动作取消。
- 输入动作值未知或 N-API 参数不合法。

拒绝结果包含稳定枚举并进入调试快照。连续点击不得绕过冷却、体力或动作窗口。

`stop()`、XComponent 销毁和显式重启必须清空待处理动作、动作状态、训练脉冲、附着、反应
窗口和表现事件，恢复配置定义的初始状态。输入采集仍不受命中停顿影响；若后续加入命中
停顿，只暂停战斗推进，不暂停输入入队。

## 6. 测试策略

### 6.1 纯 C++ 单元测试

- 四段普攻顺序、伤害、衔接窗口，以及移动、受击、超时重置。
- 体力消耗、恢复延迟、恢复上限和体力不足拒绝。
- 普通闪避、无敌边界、精准闪避和洞察只消费一次。
- 三个技能冷却、无目标和动作冲突拒绝。
- 附着刷新、过期、三种反应、顺序无关和重复触发保护。
- 共鸣普通积累、三源窗口完成/过期、终结技门槛和消费。
- 训练假人破韧、易伤、死亡和复位。
- 训练脉冲固定节奏、命中、无敌和精准窗口边界。
- 同 tick 输入和命中稳定排序，派生事件批次无递归。
- 非法配置按分组回退安全默认值。

### 6.2 集成与契约测试

- 从输入队列到固定循环，覆盖六种一次性动作和持续移动并存。
- 当前软锁定训练假人接收攻击，失效目标拒绝动作。
- 训练脉冲、角色动作、伤害、附着、反应、共鸣和快照按既定顺序更新。
- 生命周期停止与重启不残留动作、无敌、冷却、附着或脉冲。
- Node 契约验证六按钮、N-API 动作映射、单一 Native 触控生产者和新增快照字段。
- 全部既有阶段 1、阶段 2 测试继续通过。

### 6.3 构建与真机出口

- macOS 纯 C++ 测试全部通过。
- OHOS arm64-v8a `native_game` 完整编译和链接通过。
- signed HAP 构建、安装和启动通过。
- 真机通过临时按钮完成四段连击、体力耗尽/恢复、普通闪避、精准闪避、三种双源反应、
  三源连携和共鸣终结技。

## 7. 实施边界与提交策略

实施按以下依赖顺序拆分为独立可测试提交：

1. 配置、动作意图和拒绝语义。
2. 四段普攻状态机。
3. 体力、闪避、无敌与精准闪避。
4. 三源技能、附着和三种反应。
5. 共鸣积累、三源窗口和终结技。
6. 训练假人、训练脉冲与固定循环集成。
7. N-API、临时按钮、快照、渲染诊断和真机验收。

每一步先增加失败测试，再实现最小通过代码，并保持提交只包含该任务相关文件。用户本地
`build-profile.json5` 签名配置不纳入阶段实现提交，除非用户另行明确要求。
