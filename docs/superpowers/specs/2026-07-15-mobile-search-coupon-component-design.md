# 移动端搜索结果页优惠券组件设计

## 目标

在当前 Figma 文件中创建一个适用于移动端搜索结果页的迷你标签式优惠券组件。组件强调促销信息，但不应压过商品标题、价格等搜索结果核心内容。

## 组件结构

- 组件尺寸：156 × 44 px。
- 左侧金额区：展示 `¥20`，金额为组件的主要视觉焦点。
- 中部条件区：展示 `满199可用`，不展示有效期。
- 右侧操作区：根据状态展示 `领取` 或 `已领取 ✓`。
- 卡片左右边缘各设置一个半圆缺口，中间使用短虚线分隔金额信息与操作区。

## 视觉规范

- 风格：红橙促销风。
- 未领取状态：使用高饱和红橙渐变背景、白色高对比文字和白色领取按钮。
- 已领取状态：降低背景饱和度和整体对比度，保留红橙色相以维持品牌识别。
- 圆角、间距和字号应适合搜索结果页的紧凑信息密度，并保证 44 px 的可点击高度。

## 组件状态

使用一个 Figma Component Set，建立状态属性 `Status`：

- `Unclaimed`：按钮文案为 `领取`。
- `Claimed`：按钮文案为 `已领取 ✓`。

两个 Variant 保持相同尺寸与内容结构，仅改变视觉层级和操作状态，便于产品页面直接切换。

## 验收标准

- Figma 画布中存在一个命名清晰的 Component Set。
- Component Set 包含 `Unclaimed` 与 `Claimed` 两个 Variant。
- 两个状态尺寸均为 156 × 44 px。
- 文案只包含金额、使用门槛和领取状态，不包含有效期。
- 在常见移动端搜索结果卡片中，组件可作为紧凑促销标签使用。
