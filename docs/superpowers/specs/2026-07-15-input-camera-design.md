# 艾瑟兰阶段 2 输入与自由相机设计

**日期：** 2026-07-15  
**状态：** 已批准  
**依据：** `docs/superpowers/specs/2026-07-14-combat-vertical-slice-design.md`  
**阶段目标：** 双区触控、虚拟摇杆、自由第三人称相机与软锁定输入语义

## 1. 背景与范围

阶段 1 已完成线程安全输入队列、固定时间步、类型化事件、线程安全快照和 Native
生命周期管理。阶段 2 在这些接口之上建立可测试的触屏输入语义、相机相对移动、自由
第三人称相机和软锁定目标选择，为阶段 3 的角色战斗与三源技能提供稳定输入边界。

本阶段只负责输入、移动、相机和目标推荐，不实现普攻、闪避、体力、三源技能、伤害、
敌人 AI 或首领逻辑。已有攻击类 `InputAction` 可以保留，但不得在本阶段产生战斗效果。

## 2. 方案决策

采用分层输入语义方案：ArkTS 只采集多指触控并通过 N-API 传递原始指针事件；C++
负责指针区域归属、虚拟摇杆、镜头手势和规范化意图；固定 tick 消费意图并更新角色、
相机和软锁定状态。

未采用以下方案：

- ArkTS 直接计算虚拟摇杆和镜头语义：容易让输入节奏受 UI 刷新影响，并削弱确定性测试
  与未来手柄输入的统一边界。
- 在 `Loop::processInput()` 中集中实现全部行为：会继续耦合输入、玩法、相机和渲染，
  不利于阶段 3 扩展。

## 3. 架构与模块边界

```text
ArkTS 多指触控
    ↓ 原始 Pointer 事件
InputQueue
    ↓
TouchRouter
    ├─ 左区 → VirtualJoystick → MoveIntent
    └─ 右区 → CameraGesture  → LookIntent
                               ↓
固定 tick → PlayerController → ThirdPersonCamera → SoftTargeting
                               ↓
                         GameSnapshot / Renderer
```

- ArkTS 与 N-API 不解释移动、镜头和软锁定规则。
- `TouchRouter` 只维护活动指针和控制器归属。
- `VirtualJoystick` 只把左区指针位移转换为规范化移动向量。
- `CameraGesture` 只把右区指针位移累积为镜头增量。
- `PlayerController` 只计算相机相对移动，不直接读取触摸事件。
- `ThirdPersonCamera` 只维护相机轨道状态与跟随结果。
- `SoftTargeting` 只从候选目标中产生稳定的推荐结果，不控制镜头或执行攻击。
- `Loop` 编排模块并发布快照，不承载各模块的内部规则。

## 4. 多指触控与控制权

`TouchRouter` 为每个活动指针保存 `Movement`、`Camera` 或 `Ignored` 职责。

- 指针按下时根据当时的 Surface 宽度划分左右区域，之后移动越区不改变职责。
- 左半屏第一根有效手指获得移动控制权。
- 右半屏第一根有效手指获得相机控制权。
- 同一区域的额外手指标记为 `Ignored`，直到其释放或取消；本阶段不实现按钮竞争。
- `PointerUp` 和 `PointerCancel` 只释放对应指针，不影响另一区域的活动指针。
- Surface 销毁、应用停止或输入状态整体取消时，清空全部活动指针和意图。

Surface 宽高为 0、坐标不是有限值、重复按下、未知指针移动和重复释放均被安全忽略。
输入事件继续按现有 `sequence` 顺序消费。

## 5. 规范化玩家意图

固定 tick 消费的输入语义为：

```cpp
struct PlayerIntent {
  Vec2 move;
  Vec2 lookDelta;
  bool softLock;
};
```

- `move` 的每个分量位于 `[-1, 1]`，向量长度最大为 `1`。
- 虚拟摇杆以移动指针按下点为圆心，使用固定死区和最大拖动半径。
- `move` 是持续状态；没有新的移动事件时保持到对应指针释放或取消。
- `lookDelta` 累积相机指针自上次事件后的位移，并在每个固定 tick 消费后清零。
- `softLock` 是供后续操作或自动辅助使用的语义状态；阶段 2 的默认流程允许持续启用
  推荐目标，但不强制锁死镜头。
- 所有阈值和灵敏度使用强类型配置与安全默认值，不从 ArkTS 散落传入魔数。

## 6. 角色移动

`PlayerController` 将 `move` 投影到相机的水平前向和右向，得到世界空间移动方向。

- 只使用相机 yaw，不让 pitch 改变地面移动速度。
- 对角输入保持单位长度，避免斜向加速。
- 本阶段使用恒定速度和固定 `dt` 更新，不加入冲刺、加速度、体力或动作状态限制。
- 没有移动意图时停止位移，但保留角色最后有效朝向。
- 有移动意图时角色朝向平滑趋近移动方向；软锁定只提供建议朝向，不直接改写位置。
- 现有归一化灰盒场景边界继续作为阶段 2 的移动限制，暂不引入碰撞系统扩展。

## 7. 自由第三人称相机

`ThirdPersonCamera` 保存 yaw、pitch、distance、目标点和经过平滑的相机结果。

- 右区水平拖动更新 yaw，垂直拖动更新 pitch。
- pitch 和 distance 使用明确的最小值与最大值；阶段 2 不实现双指缩放，因此 distance
  使用配置默认值，但接口保留受限设置能力。
- 相机目标点跟随角色，轨道位置由目标点、yaw、pitch 和 distance 计算。
- 相机角度更新和目标跟随均基于固定 `dt`，相同输入序列产生相同结果。
- 渲染层只读取相机结果，不解释触摸事件或修改相机规则。
- 任一相机状态变成非有限值时，整体恢复为经过测试的默认 yaw、pitch 和 distance。

本阶段不实现相机遮挡探测、自动缩距、镜头震动或战斗锁定镜头。

## 8. 软锁定目标选择

`SoftTargeting` 接收玩家位置、相机水平前向和候选目标列表，输出可选目标 ID、距离和
建议朝向。

- 候选目标必须拥有有效实体 ID 和有限位置。
- 候选目标必须同时满足最大距离与相机前方夹角限制。
- 评分首先考虑与相机/视线中心的夹角，其次考虑距离。
- 评分完全相同时使用实体 ID 升序决胜，保证结果稳定。
- 当前目标失效或离开有效范围后立即清除，不保留悬空句柄。
- 没有合法候选时返回空结果。
- 阶段 2 使用现有灰盒场景目标作为候选，不生成伤害、攻击或 AI 事件。

软锁定只影响推荐目标和角色建议朝向，不强制旋转相机，不阻止玩家自由转向和脱锁。

## 9. 快照与桥接

`GameSnapshot` 在现有字段上增加阶段 2 的可诊断状态：

- 相机 yaw 与 pitch。
- 规范化移动向量。
- 软锁定目标 ID 与目标距离。

Native 层继续按现有快照发布节奏更新；ArkTS、N-API 和类型声明必须保持字段一致。
这些字段用于调试 HUD 和真机验收，正式战斗 HUD 的布局与表现仍属于阶段 6。

## 10. 错误处理与生命周期

- Surface 尺寸无效时不进行区域划分和坐标归一化，意图保持中性。
- 非有限坐标、未知指针事件和无效候选目标被安全拒绝。
- 输入队列溢出沿用阶段 1 的拒绝新事件策略，不覆盖未消费旧事件。
- Surface 销毁、应用停止和显式取消统一清除活动指针与玩家意图。
- 恢复后的第一帧不得沿用停止前的移动或镜头增量。
- 相机异常状态恢复安全默认值；软锁定异常候选直接排除。
- 异常恢复保持可诊断，但高频移动路径不得逐帧刷日志。

## 11. 自动化测试

新增纯 C++ 测试覆盖：

- `TouchRouter`：左右区域归属、多指独立释放、越区不换权、取消清理和异常输入。
- `VirtualJoystick`：死区、半径截断、对角线归一化和释放归零。
- `CameraGesture`：增量累积、逐 tick 清零和灵敏度缩放。
- `PlayerController`：相机相对移动、恒定速度、对角归一化和停止行为。
- `ThirdPersonCamera`：yaw 更新、pitch/distance 限制、固定步平滑和异常值恢复。
- `SoftTargeting`：距离/夹角过滤、稳定评分、ID 决胜和目标失效。
- `Loop`：输入队列到角色、相机、软锁定与快照的确定性集成链路。
- 桥接契约：ArkTS、N-API 与声明文件中的快照字段一致。

阶段 1 的全部 C++ 与桥接测试必须继续通过。HAP 构建不得改变已经验证的 EGL/GLES3
WindowSurface 初始化和安全销毁顺序。

## 12. 阶段验收出口

- macOS 纯 C++ 测试全部通过。
- HAP 编译通过，现有 EGL/GLES3 生命周期路径保持可用。
- 真机可同时用左手持续移动、右手环绕与俯仰镜头。
- 任一手指释放或取消不影响另一侧控制器。
- 相机无跳变、翻转或非有限状态。
- 软锁定选择稳定，目标离开有效范围后自动清除。
- 攻击、闪避和三源输入在本阶段不产生战斗效果。

满足以上出口后，才进入阶段 3“角色战斗与三源”。
