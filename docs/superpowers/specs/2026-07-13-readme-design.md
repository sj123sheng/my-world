# 项目 README 设计

## 目标

为《艾瑟兰（Ethelan）》项目提供一个兼顾开源访客与开发者的中文入口文档。读者应能在数分钟内理解项目定位、当前完成度和技术路线，并能依据文档完成本地构建与真机运行。

## 目标读者

- 希望快速了解项目玩法定位、原创特色和当前状态的访客。
- 需要在 DevEco Studio 中构建、安装和调试工程的开发者。
- 希望继续阅读架构、玩法或实施决策详情的项目协作者。

## 内容结构

README 使用以下顺序：

1. 项目名称、简短定位和当前阶段。
2. 当前能力，区分已经实现的 Native 技术骨架与长期产品愿景。
3. 技术架构，说明 ArkUI、N-API、HarmonyOS NDK C++、XComponent、NativeWindow 和 GLES3 的责任边界。
4. 环境要求，列出 DevEco Studio、HarmonyOS SDK 6.1.0(23)、HarmonyOS 设备和 C++ 工具链要求。
5. 快速开始，包括导入、同步、构建、签名、运行及命令行 Hvigor 构建方式。
6. 真机验证，说明已验证的 EGL/GLES3 运行状态和推荐日志关键字。
7. 项目目录，展示主要源码、配置、测试和设计文档位置。
8. 测试与验证，给出 fence 测试和完整 HAP 构建命令。
9. 已知限制，明确模拟器图形能力与当前 MVP 内容边界。
10. 路线图与相关文档链接。
11. 许可证状态。

## 内容边界

- README 控制在约 150 至 250 行，保持可扫描性。
- 只摘要介绍“三源共鸣”和 MVP，不复制完整玩法设计。
- 详细设计、实施计划和技术决策通过相对链接指向 `docs/superpowers/`。
- 不宣称尚未实现的大世界、联网或完整战斗内容已经可玩。
- 不写入本机 SDK 绝对路径、签名凭据或设备序列号。
- 命令以仓库根目录为执行位置，并与当前工程实际配置一致。

## 构建与验证信息

- 兼容与目标 SDK：HarmonyOS 6.1.0(23)，API 23。
- Native 图形链路：XComponent + NativeWindow + EGL 1.4 + OpenGL ES 3。
- 完整构建命令：`hvigorw assembleHap --mode module -p module=entry@default -p product=default`。
- Native fence 测试使用本地主机 C++17 编译器运行。
- 真机运行需要在 DevEco Studio 中配置调试签名。

## 验收标准

- README 中的相对链接均指向现有文件。
- 所有命令与文件路径可在当前仓库找到。
- 明确区分当前实现、设计目标和已知限制。
- 不包含敏感信息、构建产物或机器专属配置。
- Markdown 格式检查无尾随空格或空白错误。
