# M4：投资人演示版设计文档

**日期**：2026-07-21
**阶段**：M4 长期能力评审
**目标**：将现有技术骨架和战斗垂直切片打磨为面向投资方/合作方的可交互演示版

---

## 1. 背景与目标

M0-M3 已完成：HarmonyOS 原生渲染骨架（XComponent + NDK C++ + GLES3 蒙皮）、三源共鸣战斗系统、敌人 AI、首领关卡、HUD 与 VFX 均已落地并通过真机验证。

M4 不再扩展里程碑体系，而是将现有成果转化为一个可以直接展示给投资人的 demo。核心目标：让不同背景的投资人在 5-10 分钟内看到技术壁垒和产品潜力。

### 1.1 成功标准

- 投资人安装 HAP 后一键启动，无需开发者操作即可体验完整流程
- 角色对攻击/闪避/技能按钮有真实的骨骼动画反应
- 移动时角色面朝移动方向，跑步动画正确播放
- 场景有层次感，不再是灰色平面加几个模型
- 5-10 分钟流程有起承转合，包含一个"哇时刻"
- 流程结束后可进入沙盒模式自由试玩
- 技术背景投资人可看到性能监控和渲染管线深度

### 1.2 约束

- 美术资源继续使用 CC0 免费素材，M4 阶段不投入美术预算
- 不做在线服务、多人、付费系统
- 不做录制回放、截图录屏功能
- 质量优先，不设硬期限

---

## 2. 任务拆分

### Task 1：角色输入响应与移动朝向修复

**优先级**：最高（功能缺陷，阻塞后续所有任务）

**问题**：
- 点击攻击/闪避/技能按钮时角色模型没有动画反应
- 跑步时角色不能转身换方向

**修复链路**：沿"ArkTS 按钮 → Native 输入 → 战斗状态机 → 动画状态切换 → SkinnedModel clip 播放 → 渲染"逐步排查断点。

**验收**：
- 点击普攻按钮，角色播放 attack 动画
- 点击闪避按钮，角色播放 dodge 动画并有位移
- 点击技能按钮，角色播放对应技能动画
- 摇杆移动时角色面朝移动方向（yaw 旋转跟随）
- 移动时播放 run 动画，静止时播放 idle 动画
- 敌人和首领也正确播放对应动画

### Task 2：环境与场景氛围

**问题**：当前场景只有灰色地面和角色模型，视觉空旷。

**范围**（全部用程序化几何体，不依赖外部美术资源）：
- 程序化地面纹理（网格或深浅交替地砖）
- 环境物件：石柱、矮墙、边界标记（立方体/圆柱组合）
- 天空盒或渐变背景色
- 方向光 + 环境光，角色有明暗对比
- 首领区域地面配色或标记，形成视觉分区

**验收**：
- 场景不再是纯色平面，有可识别的环境结构
- 角色模型有光影明暗，不是均匀照亮
- 首领区域有视觉区分

### Task 3：完整体验流程

**结构**（5-10 分钟，每段 2-3 分钟）：

1. **开场**：角色出现在区域入口，简短文字引导，镜头从俯瞰拉近到角色身后
2. **探索**：穿过环境（石柱、矮墙），1-2 个训练假人引导尝试普攻和闪避
3. **遭遇**：进入开阔区域，2-3 个敌人出现，引导使用三源技能，HUD 提示共鸣组合
4. **首领高潮**：首领登场有镜头和画面变化，三阶段首领要求综合使用普攻+闪避+共鸣
5. **通关结算**：简短完成画面，展示用时和关键数据，回到沙盒模式入口

**沙盒模式**：从主界面可进入，自由切换敌人类型/数量/技能组合，无引导。

**验收**：
- 从启动到通关结算可在 5-10 分钟内完成
- 每段有新内容，不让人无聊
- 流程结束后沙盒模式可用

### Task 4：标志性时刻

**主"哇时刻"——共鸣爆发演出**：
- 玩家依次完成三源命中触发共鸣爆发时
- 屏幕震动、角色周围源质粒子爆发
- 敌人被击飞的慢动作瞬间
- 同时展示核心玩法机制和视觉冲击力

**辅助高光**：
- 首领登场：镜头俯瞰，地面变色扩散，首领走近，光照变化
- 韧性击破：敌人韧性归零暴露核心，聚焦发光，攻击核心时特效放大

**验收**：
- 共鸣爆发有明显的视觉冲击，投资人能记住"三源共鸣"
- 首领登场有仪式感
- 韧性击破有正反馈

### Task 5：技术亮点展示

**范围**（纯技术信号，不投入产品内容）：
- 调试模式实时显示 FPS、draw call 数、骨骼数、动画 clip 数（扩展 PerformanceGuard）
- 调试模式可切换显示骨骼绑定、网格线框、UV 检查
- 沙盒模式可选展示三源反应矩阵和敌人 AI 决策树

**验收**：
- 调试面板可切换开关
- 渲染模式切换工作正常
- 对技术背景投资人有说服力

### Task 6：一键启动与分发

**范围**：
- 安装 HAP 后启动即进入演示流程
- 沙盒模式作为流程结束后的选项出现，不阻塞主线
- 保持 HAP 体积合理，支持直接安装分发

**验收**：
- 全新安装后启动直接进入流程，无调试工具依赖
- 流程结束自动展示沙盒入口

## 3. 依赖关系

```
Task 1（输入修复）─┬─→ Task 2（环境氛围）
                   ├─→ Task 3（体验流程）
                   ├─→ Task 4（标志性时刻）
                   └─→ Task 5（技术展示）
                                         └─→ Task 6（一键启动）
```

Task 1 是阻塞项，必须最先完成。Task 2-5 在 Task 1 完成后可并行规划，但实施时按 2→3→4→5 顺序推进以保证体验连贯性。Task 6 最后收尾。

---

## 4. 风险与应对

| 风险 | 应对 |
------|------|
| 输入-动画链路修复涉及多层架构 | 用 systematic-debugging 逐层排查，不猜测 |
| CC0 资源品质上限 | 先跑通体验，后期替换美术时接口不变 |
| 流程节奏不好 | 每段限时体验，通过真机反复调整 |
| 共鸣爆发效果做不出冲击力 | 分阶段实现：先粒子，再震屏，再慢动作 |
| 场景物件影响性能 | 用基本几何体，控制 draw call 数量 |

---

## 5. 验收门槛

- 真机上完整流程可跑通，无崩溃
- 角色对全部按钮有动画反应
- 移动转身正确
- 场景有层次感和光影
- 共鸣爆发有可识别的视觉冲击
- 调试面板可切换
- 一键启动可用
--- 
## 6. 不在 M4 范围内

- 在线服务、多人、匹配
- 付费系统、内购
- 录制回放、截图录屏
- 定制美术资源
- 多区域扩展
- 云存档和跨端接续

---

## 7. 与里程碑的关系

M4 原定义为"长期能力评审"，本文档将其具体化为"投资人演示版"。演示版完成后，依据投资人反馈决定后续方向：

- 继续单机内容扩展
- 建设合作副本（独立在线服务工程）
- 扩大开放世界

此决策不在 M4 实施期间做出，而是在投资人反馈收集后评估。
